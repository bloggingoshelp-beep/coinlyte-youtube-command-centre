# CoinLyte YouTube Command Centre

Private operating system for the CoinLyte YouTube business. Helps Kirtish and the team answer daily operational questions: what to research, record, edit, and publish next; which ideas are backed by audience data, competitor gaps, or market signals; and who owns each step.

## What This App Does

- Shows a daily Command board for the next video decision, blockers, team ownership, and sponsor watch.
- Tracks YouTube analytics, latest uploads, audience comments, competitor signals, market intelligence, and AI-generated video ideas.
- Converts ideas into Content Planner cards with source links, locked research briefs, editor reference sources, assignees, target date/time deadlines, and stage movement.
- Keeps Community Pulse safer by filtering scam/reply-farm comments at both the Python refresh layer and the JavaScript frontend layer.
- Lets planner cards move with Back/Fwd controls or direct drag-and-drop into any stage, with each stage sorted by due date first by default.
- Tracks Brand Deals and sponsor records.
- Supports Team Access, app-area permissions, hashed team login codes via Supabase, role-aware in-app notifications, and optional email notifications via Resend.
- Provides owner reliability controls: planner backup/import, brand backup/import, owner settings, and logout.
- Separates lightweight board sync from the full live-data refresh.
- Runs a refresh workflow through Vercel API + GitHub Actions.

## Main App Areas

- `Command`: daily owner dashboard.
- `Analytics`: channel performance and traffic analysis.
- `Channel Intelligence`: market intel, coin stats, competitor intel, community pulse, video ideas, and action plan.
- `Content Planner`: board, saved radar, and calendar.
- `Brand Deals`: brand directory and deal board.
- `Team Access`: team users, access status, board-permission preferences, notification channels, and notification inbox.
- `Refresh`: trigger and monitor live-data refresh.

## Core Architecture

- Vercel serves all files through `api/static.js` behind login gate.
- Login uses User ID plus access code. Owner access is controlled by Vercel env vars; team access is controlled by hashed Supabase rows.
- `assets/live-data.js` is generated by GitHub Actions and contains refreshed YouTube/news/comment/AI intelligence.
- Supabase `app_state` stores the shared operating board: Planner, Brand Deals, Team Access, Saved Radar, notifications, and dismissals.
- Browser local storage remains a fallback/cache, not the long-term source of truth when Supabase is configured.
- `Refresh Live Data` rebuilds intelligence through GitHub Actions.
- The refresh workflow reads Supabase board memory before asking Claude for video ideas, so topics already planned, saved, dismissed, or recently published are blocked from regeneration.
- `Sync Board`, `Sync Planner`, `Sync Brands`, and `Sync Team` only sync operational data through Supabase — they never call YouTube, RSS, or Claude.

## Intelligence Flow

1. **Market Intel** — India Policy, US Regulation, Global Market, embedded Source Radar
2. **Coin Stats** — Top 30 CoinGecko coins by market cap, ranked by momentum score (24h×2 + 7d + volume). Coin price moves are treated as signals to find the underlying narrative — ideas explain the catalyst and sector story, not the price move.
3. **Competitor Intel** — Coin Bureau, Cyber Scrilla, Bankless upload gaps → CoinLyte-fit angles
4. **Community Pulse** — Comment demand clusters, scam/spam filtered at two layers
5. **Video Ideas** — Final ranked shortlist combining all signals. Coin momentum ideas capped at 3. Urgency covers India-breaking news, security threats, tax news, and community question spikes. Evergreen angles (Tax & Compliance, SIP & Long-Term Investing, Passive Income) are always considered alongside news signals.
6. **Content Planner** — Ideas become production cards with research brief and source links
7. **Brand Deals** — Sponsorship pipeline
8. **Team Access** — Permissions, hashed codes, notifications
9. **Refresh** — GitHub Actions pipeline trigger and status monitor

## Key Rules

- Source links must open in a new tab
- Planner cards created from intelligence must carry source links and research brief
- Saved Radar must survive refresh and can add directly to Planner
- Dismissed ideas and already-planned topics must not be regenerated
- Coin Stats must remain separate from Market Intel
- Coin momentum ideas must explain the narrative behind the price move — never raw price prediction or FOMO language
- Video Ideas must combine all signals — not duplicate any single tab
- Team access must be enforced server-side, not only hidden in the UI
- Team access codes must remain hashed (PBKDF2)
- `SUPABASE_SERVICE_ROLE_KEY` must never appear in frontend code
- All secrets stay in GitHub/Vercel environment variables only

## Data Sources

- `assets/data.js` — static seed/fallback data
- `assets/live-data.js` — refreshed by GitHub Actions (`window.COINLYTE_LIVE_DATA`)
- `assets/refresh-status.json` — last refresh metadata
- Supabase `app_state` — shared board (planner, brands, team, notifications, dismissed, saved radar)
- Supabase `team_users` — hashed team login codes and board permissions

- Zero FOMO language enforced: no 10X, pump, breakout, price targets in any title
- Evergreen minimum floors: MIN 2 Security, MIN 1 Tax & Compliance, MIN 1 SIP/Investing per refresh
