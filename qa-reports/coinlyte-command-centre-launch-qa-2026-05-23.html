Issue Found and Fixed

Notification privacy and preservation hardening. The board API already filtered operational data by app access, but notification sync needed a stronger server-side rule. Without the patch, a filtered board save could risk replacing notifications that were hidden from the current login.

  • Fixed api/board.js so notifications are filtered server-side for the current session.
  • Fixed api/board.js so hidden notifications are preserved during sync for both owner and team sessions.
  • Fixed api/notify.js so email sends are restricted to active Supabase team-user email addresses when Supabase is configured.
  • Updated automated smoke checks to protect these rules in future edits.

Backend Launch Notes

  • /api/static keeps dashboard files behind login.
  • /api/me revalidates sessions and team access against Supabase.
  • /api/board stores shared operating board state in Supabase app_state.
  • /api/team-user manages hashed team login codes.
  • /api/notify sends optional Resend emails only to active team addresses.
  • /api/refresh queues the GitHub live-data refresh workflow for owners only.

Residual Risks

  • Email should be verified in Resend logs after the next real team assignment, because delivery depends on Resend domain health and recipient mailbox filtering.
  • Production testing used the currently logged-in production session for UI checks. Owner-only destructive flows such as importing backups should still be handled deliberately.
  • News/feed quality depends on GitHub Actions refresh health and upstream YouTube/RSS availability.
